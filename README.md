# OpenMV Color Tracking Project

一个基于 OpenMV 的颜色追踪项目，支持多种颜色识别和跟踪功能。

## 项目简介

本项目使用 OpenMV 摄像头模块实现实时颜色识别和追踪。支持红色、绿色、蓝色等多种颜色的检测，并可通过串口输出目标位置信息。

## 功能特性

- 🎯 **多颜色追踪**：支持红色、黑色等多种预设颜色阈值
- 📊 **实时处理**：高效的图像处理和目标识别
- 📡 **串口通信**：通过 UART 输出目标坐标信息
- 🔧 **灵活配置**：可自定义颜色阈值和追踪参数
- 📷 **视觉反馈**：在图像上绘制目标框和十字标记

## 硬件要求

- OpenMV Cam（H7、M7 或其他兼容型号）
- USB 数据线（用于编程和调试）
- 可选：UART 串口连接设备（如 Arduino、树莓派等）

## 安装与使用

### 1. 环境准备

- 下载并安装 [OpenMV IDE](https://openmv.io/pages/download)
- 通过 USB 连接 OpenMV 摄像头到计算机

### 2. 上传程序

1. 打开 OpenMV IDE
2. 连接设备（点击左下角"连接"按钮）
3. 打开 `main.py` 或其他追踪脚本
4. 点击"运行"按钮测试，或点击"保存到 OpenMV"将程序固化

### 3. 运行追踪

```python
# 示例：红色追踪
# 运行 Track_red.py
```

程序会自动：
- 初始化摄像头
- 开始实时图像捕获
- 识别并追踪目标颜色
- 在帧缓冲区绘制检测结果
- 通过串口输出坐标信息

## 颜色阈值配置

颜色阈值使用 LAB 颜色空间定义，格式为 `(L_min, L_max, A_min, A_max, B_min, B_max)`：

```python
# 红色阈值示例
red_threshold = (30, 100, 15, 127, 15, 127)

# 黑色阈值示例
black_threshold = (0, 30, -20, 20, -20, 20)
```

### 自定义颜色阈值

1. 使用 OpenMV IDE 的"阈值编辑器"工具
2. 在实际光照条件下调整参数
3. 将生成的阈值替换到代码中

## 串口通信协议

默认串口配置：
- 波特率：115200
- 数据位：8
- 停止位：1
- 校验位：无

输出格式示例：
```
X:160 Y:120
```

其中 X、Y 为检测到的目标中心坐标。

## 性能优化建议

- 🔆 **光照条件**：确保目标区域光照充足且均匀
- 🎨 **颜色对比**：目标颜色与背景应有明显区别
- 📐 **分辨率调整**：根据需求选择合适的图像分辨率（QVGA、VGA 等）
- ⚡ **帧率控制**：平衡检测精度和处理速度

## 常见问题

### Q: 检测不到目标颜色？
A: 
- 检查光照条件
- 重新校准颜色阈值
- 确认目标大小符合 `pixels_threshold` 设置

### Q: 串口无数据输出？
A: 
- 确认 UART 引脚连接正确
- 检查波特率设置是否匹配
- 验证串口初始化代码

### Q: 帧率过低？
A: 
- 降低图像分辨率
- 减少图像处理步骤
- 优化颜色阈值以减少误检
